### OPT-003-R1

ID: OPT-003-R1  
严重程度: P1  
状态: CONFIRMED  
文件与位置: the three PluginManager PA ledger persistence adapters and `saveSettings()`  
问题机制: Store-level queues were independent. Each adapter exposed its tentative subtree on the shared settings object before an uncoordinated whole-settings save, so another successful ledger save could persist a sibling change that later failed and rolled back only in memory.  
触发条件: two ledgers persist concurrently; the first blocks and ultimately fails while the second whole-settings save succeeds.  
影响: restart could resurrect a failed ghost state despite Store snapshot rollback.  
证据: no shared adapter/settings critical section; initial tests covered only one Store or sequential failures.  
建议方向: put all PA adapter transactions and ordinary `saveSettings()` on one settings write queue, with read/apply/save/rollback inside the critical section.  
验证方式: deterministic deferred cross-adapter test captures immutable save snapshots.

Primary-agent classification: 成立.

Resolution:

- Added `settingsSaveTail` / `enqueueSettingsWrite()` shared by ordinary settings saves and all three PA ledger adapters.
- Added `persistPaSettingsSlice()` so previous read, tentative apply, save, and rollback occur inside the shared critical section.
- Added a deferred Review Queue failure versus Saved Insight success test; the successful snapshot contains no failed Review Queue item and the queue continues after rejection.
- Related post-fix gate: 7 suites, 288/288 tests; lint, type-check, build, whitespace pass.
