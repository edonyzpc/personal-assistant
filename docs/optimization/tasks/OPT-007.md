# OPT-007 — Remove obsolete mobile Debug log residue

## Pre-implementation revalidation

- Original problem: OPT-005 stopped the global Console/file logger, but old
  installations may retain `<manifest.dir>/logs.txt` containing unredacted
  cross-plugin Console output. The current test-vault plugin directory contains
  a 3,450-byte residue; only metadata was inspected.
- Root cause: the retired helper wrote exactly `${plugin.manifest.dir}/logs.txt`
  and had no cleanup or disposer.
- Confirmed user decision: on startup, automatically delete only that exact
  plugin-owned path without reading, uploading, listing, scanning, or showing a
  prompt. A failure must not block startup and may log only a fixed stage enum.
- Modification boundary: one fire-and-forget PluginManager startup helper and
  focused startup/adapter tests.
- Non-goals: no generic log cleanup, no configDir fallback, no platform/debug
  gate, no content inspection, no recovery copy, and no cleanup of other files.
- Acceptance: exact path only; missing file/dir is a no-op; exists/remove errors
  resolve without sensitive diagnostics; startup does not await cleanup.
- Residual boundary: an old hot-reload process whose retired Console closure is
  still alive can recreate the file. A cold Obsidian restart is the reliable
  final cleanup boundary.

## Implementation and initial validation

- `onload()` now starts an unawaited exact-path cleanup after settings load.
- The helper uses only `manifest.dir`, `adapter.exists`, and `adapter.remove`;
  it never reads, writes, lists, scans, uploads, or guesses a config path.
- Exists/remove failures resolve and emit only `{ stage: "exists" | "remove" }`;
  logger failure is also contained.
- Five focused assertions failed before implementation and pass afterward as
  part of `plugin-record-note.test.ts` (61 tests). Type-check, lint, build and
  whitespace pass.

## Independent gates

- Review: ACCEPT; no blocking finding. The old hot-reload Console closure is a
  disclosed pre-existing residual; cold restart is the reliable final boundary.
- Verification: PASS. Focused 63/63 and full 137 suites / 2345 tests at
  verification time plus TypeScript, lint, notices, build, bundle audit and
  diff check passed.
- Final runtime gate still must deploy/reload and confirm the exact test-vault
  residue is absent without reading it.

## Final runtime evidence

- Before cold start, only metadata for the exact test-vault residue was checked:
  `logs.txt`, 3,450 bytes. Its content was never read.
- After deploying the new bundle and performing a cold Obsidian restart, the
  exact path was absent. Plugin startup and the provider-free runner remained
  healthy.

## Rollback

- Revert the `onload()` cleanup call/helper and its focused adapter/startup
  tests. This stops future automatic deletion and restores the pre-task startup
  behavior.
- The already deleted obsolete `logs.txt` cannot and should not be reconstructed:
  it was plugin-generated diagnostic residue, not user source data, and its
  content was deliberately never read or copied.
- Re-run plugin startup tests, lint, type-check, build, bundle audit, and a cold
  test-vault start.

Status: review ACCEPT, independent verification PASS, final runtime gate PASS.
