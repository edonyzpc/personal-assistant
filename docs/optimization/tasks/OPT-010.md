# OPT-010 — Keep Vault delete statistics incremental

## Pre-implementation revalidation

- Original problem: PluginManager registers an unconditional Vault `delete`
  listener that calls `StatsManager.recalcTotals()`, while StatsManager already
  owns an incremental delete listener. Any deletion, including non-Markdown
  attachments, can therefore clear caches and reread the entire Markdown vault.
- Root cause: a legacy full-rebuild caller remained after incremental event
  ownership moved into StatsManager.
- Modification boundary: remove only the duplicate PluginManager listener and
  add production startup wiring coverage. Keep `recalcTotals()` as an explicit
  recovery API and keep all StatsManager event logic unchanged.
- Non-goals: no benchmark claim, cache redesign, refresh-delay change, VSS
  event change, or Stats UI modification.
- Acceptance: PluginManager startup registers no delete callback that invokes
  `recalcTotals`; existing StatsManager incremental delete tests remain green.

## Implementation and initial validation

- A production startup harness reproduced one `recalcTotals()` call for a
  non-Markdown delete before the fix.
- Removed only the PluginManager delete listener; StatsManager retains its
  incremental event owner and `recalcTotals()` remains available for explicit
  recovery/fallback.
- Plugin plus StatsManager tests pass 89/89; the combined module bundle, lint,
  type-check, build and whitespace pass.

## Independent gates

- Review: ACCEPT; no blocking finding.
- Verification: PASS. Baseline/current listener ownership was independently
  traced; 6 suites / 162 tests, TypeScript, lint, build and diff check passed.
- No runtime performance percentage is claimed because no reliable large-Vault
  benchmark was available.

## Rollback

- Restore only the PluginManager Vault `delete` listener that calls
  `StatsManager.recalcTotals()` and revert the matching startup regression in
  `__tests__/plugin-record-note.test.ts`.
- Leave StatsManager's incremental listeners and explicit recovery API
  unchanged. Rolling back therefore restores the former duplicate/full-rescan
  behavior but requires no data/cache migration.
- Re-run PluginManager startup and StatsManager delete tests, type-check, lint,
  build, and diff check.

Status: review ACCEPT, independent verification PASS, and final 137-suite
regression PASS. No unmeasured runtime-performance percentage is claimed.
